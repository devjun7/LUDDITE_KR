AI가 쓴 흔적을 감추는 요령보다 더 중요한 일이 있습니다. 읽는 사람이 덜 힘들게 읽고, 한 번에 뜻을 이해하도록 글을 다시 편집하는 일입니다.

**LUDDITE_KR**은 AI의 경직된 한국어, 영어 번역투, 과장된 표현, 기계적인 문단 구조를 다듬는 Agent Skill입니다. 원문의 사실과 장르는 지키면서 문장 흐름·어휘·리듬·설명 방식을 자연스럽게 바꿉니다.

Codex, Claude Code와 <code>SKILL.md</code> 형식을 지원하는 에이전트에서 사용할 수 있습니다. 저장소 이름과 표시 이름은 <code>LUDDITE_KR</code>이지만, 규격에 맞춘 내부 호출 이름은 <code>$luddite-kr</code>입니다.

## 30초 요약

**문제:** AI 초안은 뜻이 맞아도 한국어답지 않을 때가 많습니다. 주어와 대명사를 되풀이하고, <code>~을 통해</code>와 <code>~에 대해</code>를 겹쳐 쓰며, 모든 문단을 비슷한 길이와 구조로 정리합니다. 어려운 내용일수록 추상 명사와 과장된 수식어가 늘기도 합니다.

**해결:** LUDDITE_KR은 먼저 독자와 글의 목적을 파악합니다. 그런 다음 정보의 순서를 바로잡고, 문제가 있는 문장만 고칩니다. 마지막에는 의미가 달라지지 않았는지와 한 번에 읽히는지를 따로 확인합니다.

가장 중요한 기준은 하나입니다.

> 사람이 편하게 읽고 쉽게 이해할 수 있는가?

~~~

이미 같은 경로가 있다면 기존 폴더나 심볼릭 링크를 확인한 뒤 설치하세요.

## 무엇을 바꾸는가

| 기준 | LUDDITE_KR이 하는 일 |
|---|---|
| 읽기 쉬움 | 핵심을 앞에 두고, 한 문장에 중심 생각을 하나만 남깁니다. |
| 의미 보존 | 수치·날짜·이름·인용·조건·예외·확신의 정도를 불변값으로 다룹니다. |
| 한국어다운 문장 | 영어식 대명사, 긴 관형절, 겹조사, 불필요한 피동과 명사화를 줄입니다. |
| 자연스러운 흐름 | 독자가 궁금해할 순서에 맞춰 문단을 배열하고 필요한 관계만 연결합니다. |
| 장르 유지 | 보고서를 에세이로, 칼럼을 홍보문으로 바꾸지 않습니다. |
| 절제된 표현 | <code>혁신적</code>, <code>획기적</code>, <code>무한한 가능성</code> 같은 말은 근거가 없으면 걷어냅니다. |
| 오피니언 구성 | 논지, 근거, 해석, 강한 반론, 결말이 한 줄기로 이어지게 합니다. |
| 과윤문 방지 | 자연스러운 구간은 남기고, 수정량이 커지면 의미와 장르부터 다시 확인합니다. |

맞춤법 검사기처럼 단어만 바꾸지 않습니다. 문장의 어색함이 정보 순서에서 생겼다면 순서를 고치고, 설명 한 단계가 빠졌다면 그 빈자리를 알려줍니다. 근거가 없는데 사실을 만들어 채우지는 않습니다.

## 사용 예시

### 기본 윤문

~~~text
$luddite-kr 아래 글을 표준 강도로 윤문해줘.
뜻, 숫자, 고유명사, 인용은 그대로 두고 자연스러운 한국어로 바꿔줘.

[본문]
~~~

### 최소 수정

~~~text
이 문서는 원문 어조를 최대한 보존해줘.
명백한 번역투와 어색한 연결만 고치고 구조는 건드리지 마.
~~~

### 강한 수정

~~~text
핵심 주장과 근거는 유지하되 문단 순서부터 다시 잡아줘.
비전문가도 한 번에 이해할 수 있는 설명문으로 써줘.
~~~

### 신문 오피니언

~~~text
이 글을 800자 안팎의 오피니언으로 다듬어줘.
논지를 초반에 드러내고, 근거가 왜 주장을 뒷받침하는지 설명해줘.
가장 강한 반론도 공정하게 다뤄줘.
~~~

## 전과 후

수정 전:

> 오늘날 정보의 양이 급격히 증가함에 따라 효과적인 정보 관리의 중요성이 더욱 강조되고 있습니다. 이러한 맥락에서 본 시스템은 혁신적인 기능을 통해 사용자 편의성을 극대화하는 데 기여할 수 있습니다.

수정 후:

> 정보가 빠르게 늘면서 제대로 관리하는 일이 더 중요해졌다. 이 시스템은 관리 과정을 단순하게 만들어 사용자의 수고를 줄인다.

수정 전:

> 문제 해결을 위한 추가적인 논의가 진행될 필요가 있다.

수정 후:

> 문제를 풀려면 더 논의해야 한다.

짧게 만드는 것 자체가 목표는 아닙니다. 조건과 예외가 중요한 문장은 필요한 만큼 길게 씁니다.

## 작동 원리

1. **상황 파악**
   글의 목적, 독자, 매체, 장르, 격식 수준을 먼저 판단합니다.

2. **진단**
   전체 흐름부터 문장 연결, 구조, 어휘, 목소리, 문장부호 순으로 살핍니다.

3. **흐름 정리**
   설명은 핵심 답부터, 주장은 논지부터, 안내는 독자가 할 일부터 놓습니다.

4. **국소 편집**
   번역투·중복·과장·기계적 리듬이 나타난 구간만 고칩니다.

5. **이중 검수**
   먼저 의미 보존을 확인하고, 그다음 독해와 자연스러움을 확인합니다.

자세한 판단 기준은 [SKILL.md](./SKILL.md)에 모두 공개되어 있습니다. 외부 API, 실행 파일, 텔레메트리는 없습니다. 에이전트가 로컬의 Markdown 지침을 읽어 적용합니다.

## 지키는 선

LUDDITE_KR은 다음을 하지 않습니다.

- AI 탐지기를 반드시 통과한다고 약속하지 않습니다.
- 사람처럼 보이게 하려고 오탈자나 잘못된 띄어쓰기를 넣지 않습니다.
- 원문에 없는 경험, 감정, 수치, 인용, 사례를 만들지 않습니다.
- 법률·의료·재무·연구 문서의 정확성을 자연스러움과 맞바꾸지 않습니다.
- 특정 개인의 말투를 허락 없이 복제하지 않습니다.
- 이미 자연스러운 문장을 수정량을 채우려고 다시 쓰지 않습니다.

이 스킬은 편집 원칙입니다. 사실 검증, 맞춤법 전수 검사, 법률 검토를 대신하지 않습니다.
